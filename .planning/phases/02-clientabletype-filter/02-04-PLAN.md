<task type="auto">
  <name>Task 2: Create _clientable_types.html.erb partial</name>
  <files>app/views/filters/settings/_clientable_types.html.erb</files>
  <action>
  Create new file app/views/filters/settings/_clientable_types.html.erb

  Follow the exact pattern from _states.html.erb which uses the `combobox` controller for single-selection:

  ```erb
  <% filter = user_filtering.filter %>

  <%%= tag.div class: "quick-filter",
        data: {
          controller: "dialog filter combobox",
          action: "keydown.esc->dialog#close click@document->dialog#closeOnClickOutside dialog:close@document->filter#clearInput",
          filter_show: user_filtering.show_clientable_types?,
          combobox_default_label_value: "Tipo cliente..." } do %>
    <button type="button" class="btn input input--select flex-inline txt-x-small" data-action="click->dialog#toggle:stop">
      <span class="overflow-ellipsis" data-combobox-target="label">
        <%%= filter.clientable_type.present? ? user_filtering.clientable_types_disponibili[filter.clientable_type] : "Tipo cliente..." %>
      </span>
    </button>

    <template data-combobox-target="hiddenFieldTemplate">
      <%%= hidden_field_tag :clientable_type, nil, data: { filter_settings_target: "field" } %>
    </template>

    <%%= filter_dialog "Tipo cliente..." do %>
      <strong class="popup__title">Filtra per tipo cliente...</strong>

      <ul class="popup__list" data-filter-target="list" role="listbox">
        <%% user_filtering.clientable_types_disponibili.each do |type_key, type_label| %>
          <%%= tag.li class: "popup__item", data: {
                navigable_list_target: "item", combobox_target: "item", combobox_value: type_key, combobox_label: type_label },
                role: "checkbox", aria: { checked: filter.clientable_type == type_key } do %>
            <button type="button" class="btn popup__btn" data-action="dialog#close combobox#change filter-settings#change form#submit">
              <span class="overflow-ellipsis flex-item-grow"><%%= type_label %></span>
              <%%= icon_tag "check", class: "checked flex-item-justify-end" %>
            </button>
          <%% end %>
        <%% end %>
      </ul>
    <%% end %>
  <%% end %>
  ```

  Key elements matching _states.html.erb pattern:
  - Uses `combobox` controller (NOT single-selection-combobox which does not exist)
  - Uses `data-combobox-default-label-value` for the no-selection label
  - Uses `data-combobox-target="label"` on the span for dynamic label updates
  - Uses `<template data-combobox-target="hiddenFieldTemplate">` for the hidden field
  - Uses `data-combobox-target="item"`, `data-combobox-value`, `data-combobox-label` on list items
  - Iterates with |type_key, type_label| since clientable_types_disponibili returns a hash
  </action>
